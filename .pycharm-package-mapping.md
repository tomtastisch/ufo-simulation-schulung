# PyCharm Package-Mapping Hinweise

## Problem: Falsche Warnung bei PyQt5-Import

PyCharm zeigt in `ufosim.py` Zeile 178 folgende **irreführende** Warnung:

```
Package containing module 'PyQt5' is not listed in the project requirements
Please add "python-qt5" package to requirements...
```

## Warum ist diese Warnung FALSCH?

1. **`PyQt5` ist korrekt installiert** (siehe `pip list`):
    - PyQt5==5.15.11
    - PyQt5-Qt5==5.15.18
    - PyQt5-sip==12.17.1

2. **`python-qt5` ist ein ANDERES, veraltetes Paket**:
    - Nur für Windows-Binaries
    - Verursacht Build-Fehler auf macOS/Linux
    - Wird von diesem Projekt NICHT verwendet

3. **requirements.txt enthält korrekt `PyQt5`** (nicht `python-qt5`)

## Korrekte Package-Zuordnung

| Import-Statement        | Korrektes PyPI-Paket | Falsches Paket (nicht verwenden) |
|-------------------------|----------------------|----------------------------------|
| `from PyQt5 import ...` | **PyQt5** ✅          | python-qt5 ❌                     |

## Wie du die Warnung ignorierst/behebst

### Option 1: Warnung in PyCharm unterdrücken (empfohlen)

1. Klicke auf die gelbe Glühbirne bei der Warnung
2. Wähle "Suppress for statement" oder "Suppress for file"
3. Alternativ: Rechtsklick → "Suppress Inspection" → "Package requirements"

### Option 2: PyCharm-Cache invalidieren

1. Menü: File → Invalidate Caches...
2. Wähle "Invalidate and Restart"
3. PyCharm startet neu und baut den Index neu auf

### Option 3: Warnung per Kommentar unterdrücken

Füge über der Import-Zeile hinzu:

```python
# noinspection PyPackageRequirements
from PyQt5 import QtWidgets, QtGui, QtCore
```

## Technischer Hintergrund

PyCharm verwendet eine interne Datenbank zur Zuordnung von Import-Namen zu PyPI-Paketnamen.
Diese Zuordnung ist in diesem Fall **veraltet oder fehlerhaft**.

- **Korrekt:** `PyQt5` (Import-Name) → `PyQt5` (PyPI-Paket)
- **Falsch:** `PyQt5` (Import-Name) → `python-qt5` (PyPI-Paket) ❌

## Verifizierung

Prüfe selbst, dass PyQt5 korrekt funktioniert:

```bash
# Im Projekt-venv
./.venv/bin/python -c "from PyQt5 import QtWidgets; print('PyQt5 OK')"
```

Erwartete Ausgabe: `PyQt5 OK` (ohne Fehler)

---

**Fazit:** Die Warnung ist ein IDE-Bug. Ignoriere sie oder unterdrücke sie. Füge **AUF KEINEN FALL** `python-qt5` zu den
requirements hinzu!
